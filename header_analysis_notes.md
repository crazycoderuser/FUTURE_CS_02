# Email Header Analysis Notes — Sample 01

**Tool used:** Google Admin Toolbox Message Header Analyzer + MXToolbox  
**Sample:** sample_01_high_risk.txt  
**Analysis date:** April 2025

---

## Received Chain (Read Bottom-Up)

The email passed through two hops before delivery. The originating server was `45.33.32.156`, which resolves to a host in Eastern Europe — geographically inconsistent with the claimed identity of a US-based security team.

| Hop | Server | IP | Notes |
|-----|--------|----|-------|
| 1 (origin) | mail.secure-account-verify.com | 45.33.32.156 | Eastern Europe — suspicious |
| 2 (delivery) | mx.company.com | — | Recipient's mail server |

---

## Authentication Results

| Protocol | Result | Meaning |
|----------|--------|---------|
| SPF | **FAIL** | The originating server (45.33.32.156) is not listed as an authorized sender for the domain `secure-account-verify.com`. The email is not from who it claims. |
| DKIM | **FAIL** | The email's cryptographic signature is absent or invalid. The message cannot be verified as unaltered. |
| DMARC | **FAIL** | The domain's policy has been violated. A `p=NONE` policy means no automatic rejection occurred, but the failure is still a confirmed red flag. |

**Verdict:** All three authentication mechanisms fail. This email is not from an authenticated or trusted sender.

---

## Key Field Anomalies

**Return-Path mismatch.** The `Return-Path` header shows `bounce@secure-account-verify.com` — consistent with the spoofed domain, confirming this is a purpose-built phishing infrastructure and not a misconfigured legitimate server.

**Reply-To mismatch.** The `Reply-To` field directs any response to `collect@secure-account-verify.com` — a different address to the `From:` field. This is a classic indicator that the attacker wants replies (and any credential submissions) sent to a collection address separate from the sending infrastructure.

**X-Mailer field.** The `X-Mailer: PhishKit/2.1` header is a signature left by phishing toolkit software. Legitimate email clients (Outlook, Gmail, Apple Mail) use their own identifiers. This string confirms the email was generated by automated phishing infrastructure.

**Message-ID domain.** The Message-ID (`20250410.021400.phish1@secure-account-verify.com`) uses the same fake domain, consistent with the rest of the phishing infrastructure.

---

## Summary

This header analysis confirms that the email originated from an unauthorized server in Eastern Europe, failed all three email authentication protocols, and was generated by automated phishing toolkit software. The email is not from a legitimate sender under any interpretation of the header data.
